# Project Context

## Project Name

Balesin AI

## Current Objective

Create the initial professional project foundation for a Next.js application based on the product concept in `konsep project.md`.

## Product Direction

- Product type: SaaS dashboard plus marketing-ready landing experience
- Core niche: Indonesian SMBs, starting from workshops and service businesses
- Value proposition: omnichannel AI customer service with knowledge-grounded replies and human handoff

## Current Phase

Foundation and documentation setup.

## Selected Architecture Direction

- Frontend: Next.js App Router with React and TypeScript
- Styling: Tailwind CSS v4 with CSS variable based design tokens
- State management: local component state for now
- Data layer: placeholder service-friendly architecture, no premature global state
- UI system: reusable primitives and section-based composition

## Feature Priorities

1. Professional project scaffold
2. Product specification and engineering governance files
3. Reusable UI foundation
4. Dashboard/landing starter experience
5. Readiness for future API, auth, and database integrations

## Constraints

- No backend implementation in this iteration
- No hardcoded secrets
- Keep the codebase small but scalable
- Avoid overengineering before real data and auth requirements are confirmed

## Notes For Future Iterations

- Add auth provider and tenant model
- Add API layer and server actions where justified
- Add persistence layer and schema
- Add live inbox, AI tester, and knowledge base upload flow
- Add component tests and end-to-end tests
